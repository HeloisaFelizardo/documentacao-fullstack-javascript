Introdução

Nessa documentação apresentamos como criar uma API RESTFul utilizando o Node.js e um cliente SPA em React. Ao concluí-la você conhecerá alguns dentre os principais frameworks comumente utilizados em conjunto dessa tecnologia, bem como o passo a passo para criar aplicações utilizando boas práticas.

Conhecendo a aplicação

A API RESTful que criaremos passo a passo nessa documentação é destinada ao cadastro de spoilers. Ela contará com a implementação das operações de CRUD sobre uma entidade e também recursos fundamentais como validação e paginação.

Assim como já pode ser deduzido, o principal dado que será manipulado na aplicação é o Spoiler, o qual modelamos conforme o exemplo a seguir:

[curly_open]
    “id”: 1,
    "titulo": "Hereditário",
    "espoliador": "edrd",
    "descricao": "Charlie é Paimon."
[curly_close]

Para permitir que esse dado seja persistido, criaremos ao redor dele um controller, um model e um repositório. O controller será acessado a partir de um sistema de roteamento, que fará a conexão entre as URIs utilizadas para identificar o recurso spoiler e o método do controller que processará a requisição. No controller, usaremos o repositório para realizar a busca de spoilers, usando paginação quando necessário, bem como para incluir, editar e excluir esse recurso no servidor.

Na tabela abaixo temos os endpoints utilizadas pelos clientes para se conectarem a API:

Método endpoint
POST localhost:3000/api/spoilers
GET localhost:3000/api/spoilers/?limite=1&pagina=0
GET localhost:3000/api/spoilers/1
GET localhost:3000/api/spoilers
PUT localhost:3000/api/spoilers/1
DELETE localhost:3000/api/spoilers/1

Tecnologias utilizadas

Usaremos nessa aplicação o Node.js na versão 8 ou posterior. Como framework para a criação da API usaremos o Express, um dos mais utilizados pela comunidade de programadores no desenvolvimento de aplicações web em geral, incluindo APIs RESTful.

Como banco de dados usaremos o MySQL, um banco de dados relacional amplamente utilizado. Em conjunto, aprenderemos como acessá-lo utilizando o ORM Sequelize, que suporta os bancos de dados relacionais mais utilizados atualmente como SQL Server, PostgreSQL, SQLite e o MySQL.

Configurando o projeto

Graças ao NPM, a configuração de um projeto Node é simplificada, de modo que podemos criar um arquivo chamado package.json com o conteúdo abaixo em um diretório qualquer e invocar o comando npm install nesse diretório:

[curly_open]
  "name": "spoiler",
  "version": "1.0.0",
  "private": true,
  "scripts": [curly_open]
    "start": "node ./src/app.js"
  [curly_close],
  "dependencies": [curly_open]
    "express": "^4.16.3",
    "http-status": "^1.2.0",
    "mysql2": "^1.6.1",
    "sequelize": "^4.39.0"
  [curly_close]
[curly_close]

Ao fazer isso, serão lidas as dependências do projeto e um a um seus pacotes serão adicionados ao diretório node_modules.
